# GrooveForge Release Readiness Evidence

This matrix is the current proof trail for the local-first GrooveForge MVP. It does not replace QA commands; it explains which current files and gates support each completion requirement.

## Release Gate

Run the local release gate before a completion report:

```sh
npm run release:check
```

Run the compact completion progress report when the user needs a fresh progress update:

```sh
npm run release:progress
```

Run the fast release doctor before private-input or channel setup work:

```sh
npm run release:doctor
```

Prepare the ignored local distribution env scaffold before filling operator-owned private values:

```sh
npm run release:prepare-env
```

Rerun the fast redacted external preflight after editing the ignored local env file:

```sh
npm run release:external-preflight
```

Generate the compact next-actions view after external preflight, or bootstrap missing source evidence:

```sh
npm run release:next-actions
```

`release:check` runs `npm run qa` and `npm run verify`. Together those commands cover documentation/base rules, quality gate checks, first-run React renderer smoke, first-session workflow smoke, runtime export smoke, local delivery package artifact smoke, local package reopen artifact smoke, TypeScript contracts, production build, Electron desktop entry smoke, live desktop visual launch smoke, native desktop project IO smoke, packaged desktop app smoke, packaged project IO smoke, local ad-hoc signing smoke, hardened runtime readiness smoke, local DMG smoke, local PKG smoke, local PKG payload launch smoke, local PKG payload project IO smoke, simulated install-path smoke, installed project IO smoke, Gatekeeper readiness smoke, release artifact manifest smoke, release notes artifact smoke, support artifact smoke, update feed config smoke, update metadata policy smoke, update metadata artifact draft smoke, auto-update readiness smoke, Developer ID readiness smoke, Developer ID signing smoke, notarization smoke, notarized Gatekeeper smoke, distribution manual QA checklist smoke, distribution-channel QA smoke, distribution handoff smoke, distribution bundle manifest smoke, distribution env template smoke, release prepare-env smoke, distribution private-inputs smoke, completion audit smoke, external distribution gate dry-run smoke, external remediation smoke, completion status smoke, external operator runbook smoke, external readiness ledger smoke, completion progress smoke, release external preflight, and release next-actions smoke.

`release:progress` runs `npm run release:check`, reads the regenerated completion progress artifact, then writes a compact release progress Markdown/JSON report under ignored `build/desktop/` without recording private values or claiming external distribution completion.

`release:prepare-env-smoke` writes a value-free prepare-env Markdown/JSON artifact plus an ignored build scaffold from `harness/templates/distribution-private-inputs.env.example` without touching root `.env.distribution.local`. The scaffold groups distribution metadata, manual QA approval, update feed/channel, Developer ID identity, and notarization credentials with local-only guidance and rerun commands. `release:prepare-env` creates ignored `.env.distribution.local` only when the operator explicitly requests it and refuses to overwrite an existing file unless rerun with `--force`.

`release:doctor` runs targeted redacted distribution input/readiness smokes, reads the generated env-template, prepare-env, update-feed, Developer ID readiness, manual QA, distribution-channel QA, and private-input summaries, then writes a compact release doctor Markdown/JSON report under ignored `build/desktop/` without running the full release gate, recording private values, probing remote channels, signing artifacts, submitting to Apple, or claiming external distribution completion.

Release doctor reporting is targeted and value-free. Release doctor report runs targeted redacted distribution input/readiness smokes, includes prepare-env scaffold readiness, and records only key names, readiness booleans, source artifact paths, counts, and blockers.

`release:external-preflight` runs the fast redacted doctor plus completion audit, external distribution gate dry-run, external remediation, completion status, external operator runbook, external readiness ledger, and completion progress smokes, then writes an external preflight Markdown/JSON report under ignored `build/desktop/`. It is meant for the operator loop after `release:prepare-env` and local env edits; it reports hard-gate would-fail posture, requirement counts, remediation counts, private-input group counts, source evidence, and first blockers without running the full release gate, recording private values, probing remote channels, signing artifacts, submitting to Apple, or claiming external distribution completion.

`release:next-actions` reruns the external preflight when source evidence exists; if ignored source evidence is missing, it still writes a bootstrap external next-actions Markdown/JSON report under ignored `build/desktop/` with `bootstrapMode`, `sourceEvidenceReady: false`, missing source artifact rows, and first action `npm run release:check`. If release channel metadata is blocked and no local env file is loaded, it makes `npm run release:prepare-env` the first release-channel command so the ignored `.env.distribution.local` scaffold is created before private-input validation. If a local env file is loaded but placeholders remain, it reports local env placeholder key names/counts, makes placeholder cleanup the first release-channel blocker, and makes `npm run release:doctor` the next command after editing. In both paths it surfaces the current focus, prioritized pending remediation groups or the evidence prerequisite, next commands, first blockers, source evidence paths, and the hard-gate command without recording private values, probing remote channels, signing artifacts, submitting to Apple, or claiming external distribution completion. `release:next-actions-smoke` remains the strict release-gate path that reads existing preflight evidence for `npm run verify`.

`release:external-check` is the hard external distribution gate. It runs `npm run release:check`, then fails until completion audit, desktop project IO evidence including PKG payload project IO, private inputs, distribution-channel QA, auto-update, Developer ID signing, notarization/stapling, and notarized Gatekeeper evidence all prove external distribution readiness.
