# GrooveForge Release Readiness Evidence

This matrix is the current proof trail for the local-first GrooveForge MVP. It does not replace QA commands; it explains which current files and gates support each completion requirement.

## Release Gate

Run the local release gate before a completion report:

```sh
npm run release:check
```

`release:check` runs `npm run qa` and `npm run verify`. Together those commands cover documentation/base rules, quality gate checks, first-run React renderer smoke, first-session workflow smoke, runtime export smoke, TypeScript contracts, production build, Electron desktop entry smoke, live desktop visual launch smoke, packaged desktop app smoke, local ad-hoc signing smoke, hardened runtime readiness smoke, local DMG smoke, simulated install-path smoke, Gatekeeper readiness smoke, release artifact manifest smoke, auto-update readiness smoke, and Developer ID readiness smoke.

## Not Claimed By This Gate

- PKG installer creation, real `/Applications` install, Developer ID signing, hardened runtime completion, notarization, Gatekeeper approval, auto-update, app-store submission, and external distribution-channel QA are not claimed until a distribution target, developer identity, notary credentials, and update provider are selected. The gate does claim a local macOS portable `GrooveForge.app` package smoke, a local ad-hoc signing smoke with hardened runtime option and entitlement evidence, a hardened runtime readiness smoke that reports local `codesign` blockers without claiming notarization readiness, a local macOS DMG smoke, a simulated install-path copy-and-launch smoke under ignored `build/desktop/`, a Gatekeeper readiness smoke that reports local `spctl` blockers without claiming approval, a local release artifact manifest with checksums, ad-hoc signing evidence, hardened runtime flag and entitlement evidence, explicit Developer ID/notarization-unclaimed flags, an auto-update readiness smoke that reports missing updater/feed/signed-update behavior without probing a network feed, and a Developer ID readiness smoke that reports missing external-distribution prerequisites without making network submissions.
- A visible Electron GUI launch is available through `npm run desktop`, but the automated release gate uses renderer SSR smoke, `desktop:smoke`, and a hidden-window `desktop:launch-smoke` with screenshot pixel evidence instead of requiring manual window inspection.
- Real browser download shelf inspection is not claimed; the current gate verifies the browser download contract with mocked DOM and URL APIs.
- Optional sampling, audio import, sample chopping, sampler tracks, plugin hosting, cloud sync, remote AI, payments, analytics, and accounts remain outside the current MVP.

## Completion Use

Completion reports should cite `npm run release:check` plus any target-specific checks that were added after this matrix. If a future plan changes the MVP definition, export targets, desktop distribution target, or optional sampling scope, update this matrix in the same plan.
