# GrooveForge Release Readiness Evidence

This matrix is the current proof trail for the local-first GrooveForge MVP. It does not replace QA commands; it explains which current files and gates support each completion requirement.

## Release Gate

Run the local release gate before a completion report:

```sh
npm run release:check
```

```sh

```

Run the one-command completion summary refresh after each completed work before reporting the user-facing completion percentage:


That receipt runs the progress refresh first, writes the compact completion summary readout, records the value-free `npm run release:channel-apply-private-env-preflight` private-input readiness check before the `npm run release:channel-apply-private-env` write helper and strict proof chain, mirrors the separate `npm run release:channel-apply-private-env-proof` convenience runner after preflight readiness, refreshes the external completion run/resume packets from that just-refreshed summary without recursion, mirrors the preflight Process Env Input Checklist through the operator completion brief, records explicit `userFacingCompletion...` and `userFacingRemaining...` aliases plus value-free git branch/HEAD/worktree/dirty-count context without git status paths, and automatically runs the 10-plan checkpoint when the refreshed completed-plan window is at `10/10`.
